What is SimpleElastix?

SimpleElastix is an image processing library that makes state-of-the-art medical image registration really easy to do. With SimpleElastix you can access fast C++ registration algorithms from your favorite language like Python, Ruby, Java and R.

Designed For Researchers

A user-friendly API allows you to quickly prototype an application, write robust reusable components and let SimpleElastix handle the low-level computation.

Multilingual

Native bindings for C++, Python, Java, R, Ruby, Octave, Lua, Tcl and C# allows you to use state-of-the-art algorithms together with your language of choice.

Powerful

Efficient multi-threaded C++ components gets you fast results and accelerates development. You can further tune GPU components and stochastic sub-sampling for even more speed.

Parameter File Interface

A parameter-driven interface makes configuration of elastix as simple as setting string variables. Pre-configured parameter files for rigid-, affine-, non-rigid- and non-rigid groupwise registration serve as good starting points for new users.

Easy To Extend

Elastix allows you to easily write your own transforms, metrics, optimizers etc. and call your own C++ code in any of the supported languages.

SimpleITK Integration

SimpleElastix comes with the entire library of SimpeITK image processing algorithms and works directly on these images. This allows you to pre- and post-process images with robust SimpleITK algorithms.

Example 1: A Single Line of Code

With SimpleElastix it is dead simple to align images of different patients. Depending on the complexity of your problem, you can often register images with a single function call (in this case Python).


 >>> resultImage = SimpleITK.Elastix(fixedImage, movingImage, "nonrigid")

Example 2: Robust Initialization

Difficult registration problems often need initialization to account for global translation, scaling and rotation prior to non-rgid registration. The registration method is configured by an intuitive parameter interface. We pass elastix a vector of parameter maps and elastix will apply the different types of registrations sequentially using the output of one registration as input for the next.

    
 >>> parameterMapList = sitk.ParameterMapList()
 >>> parameterMapList.append(sitk.GetDefaultParameterMap("translation"))
 >>> parameterMapList.append(sitk.GetDefaultParameterMap("rigid"))
 >>> parameterMapList.append(sitk.GetDefaultParameterMap("affine"))
 >>> parameterMapList.append(sitk.GetDefaultParameterMap("nonrigid"))
 >>> resultImage = sitk.Elastix(fixedImage, movingImage, parameterMapList)

Example 3: Atlas-based Segmentation

Using an object-oriented interface you can obtain transform parameters from one registration and apply them to other images. For example, you can transform a label image to do atlas-based segmentation.


 # Non-rigid registration
 selx = SimpleITK.SimpleElastix()
 selx.SetParameterMap(sitk.GetDefaultParameterMap())
 selx.SetFixedImage(fixedImage)
 selx.SetMovingImage(movingImage)

 # We turn off interpolation for labels
 parameterMap = selx.GetTransformParameterMap()
 parameterMap[0]["ResampleInterpolator"] = ["FinalNearestNeighborInterpolator"]

 # Warp label image using transform from above
 resultLabel = SimpleITK.Transformix(movingLabels, parameterMap)
