What is SimpleElastix?

SimpleElastix is a fork of SimpleITK that integrates the elastix C++ image registration library. This makes state-of-the-art medical image registration really easy to do in languages like Python, Java, Ruby and R.

Designed For Researchers

A user-friendly API facilitates rapid prototyping, reusable components and reduces the overhead if processing large data sets.

Multilingual

Use your favorite programming language. SimpleElastix comes with native bindings for C++, Python, Java, R, Ruby, Octave, Lua, Tcl and C#.

Powerful

Efficient multi-threaded C++ components produces fast results and accelerates development. Component parameters can be tuned for even more speed.

Parameter-driven Configuration

A parameter map interface makes it easy to configure registration components. Try out different transforms, metrics and optimizers by setting string variables.

Quickstart

Pre-configured parameter files for rigid-, affine-, non-rigid- and non-rigid groupwise registration serve as good starting points for new users.

Easy To Extend

Write your own transforms, metrics and optimizers in elastix and use SimpleElastix to call your C++ code in any of the supported languages.

Example 1: A Single Line of Code

With SimpleElastix it is dead simple to align images of different patients. Depending on the complexity of your problem, you can often register images with a single function call since elastix uses a multi-resolution strategy with Adaptive Stochastic Sampling (Klein et al. 2009) and Transform Bending Energy regularisation (Klein & Staring 2010) by default. The following is Python code.


 >>> fixedImage = SimpleITK.ReadImage("fixedImage.nii")
 >>> movingImage = SimpleITK.ReadImage("movingImage.nii")
 >>> resultImage = SimpleITK.Elastix(fixedImage, movingImage, "nonrigid")

Example 2: Robust Initialization

Difficult registration problems often need initialization to account for global translation, scaling and rotation prior to non-rigid registration. You can pass a vector of configuration objects (which we refer to as "Parameter Maps") and have elastix run registrations sequentially using output of one registration as input for the next.

    
 >>> parameterMapList = SimpleITK.ParameterMapList()
 >>> parameterMapList.append(SimpleITK.GetDefaultParameterMap("affine"))
 >>> parameterMapList.append(SimpleITK.GetDefaultParameterMap("nonrigid"))
 >>> resultImage = sitk.Elastix(fixedImage, movingImage, parameterMapList)

Example 3: Atlas-based Segmentation

You can apply transformations to multiple images. For example, you can transform a label image for atlas-based segmentation.


 # Non-rigid registration
 elastix = SimpleITK.SimpleElastix()
 elastix.SetParameterMap(sitk.GetDefaultParameterMap())
 elastix.SetFixedImage(fixedImage)
 elastix.SetMovingImage(movingImage)
 elastix.Execute()

 # We turn off interpolation for labels
 parameterMap = elastix.GetTransformParameterMap()
 parameterMap[0]["ResampleInterpolator"] = ["FinalNearestNeighborInterpolator"]

 # Warp label image using transform from above
 resultLabel = SimpleITK.Transformix(labelImage, parameterMap)

Example 4: Configuring Registration Components

Elastix introduced the concept of a "parameter map" to specify registration components. A parameter map is a collection of key-value pairs that completely configure the algorithms and any settings they might require. Here is a couple of examples.


 parameterMap = SimpleITK.GetDefaultParameterMap("nonrigid")

 # Want to monitor registration convergence?
 parameterMap["WriteResultImageEachIteration"] = ["true"]

 # Want to use another optimizer?
 parameterMap["Optimizer"] = ["QuasiNewtonLBFGS"]

 # Want a more smooth deformation field?
 parameterMap["Metric1Weight"] = ["2.0"]

 # Want a more dense b-spline deformation field (mm)?
 parameterMap["FinalGridSpacingInPhysicalUnits"] = ["4.0"]

 # Want to run registration faster?
 parameterMap["NumberOfResolutions"] = ["1"]
 parameterMap["NumberOfIterations"] = ["128"]
 parameterMap["NumberOfSamples"] = ["1024"]

 # Want to write smaller images to disk?
 parameterMap["ResultImagePixelType"] = ["short"]

 # Want the raw b-spline parameters?
 bsplineParameters = elastix.GetTransformParameterMap()[0]["TransformParameters"]

 # Run registration with new parameters
 resultImage = SimpleITK.Elastix(fixedImage, movingImage, parameterMap)
